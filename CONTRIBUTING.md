# Contributing Guidelines

## How to Contribute a New Project

1. Fork the repository
2. Create `projects/XXX-project-name/`
3. Follow the project template
4. Ensure project focuses on LEARNING, not solutions
5. Include all 8 required sections
6. Submit pull request

## What We Look For

✅ Learning-focused (not solution-focused)
✅ Clear learning outcomes
✅ Thorough architecture guide
✅ Database design questions
✅ Complete pseudocode
✅ Common mistakes section
✅ Debugging strategies
✅ Testing approach

## What We Reject

❌ Copy-paste ready code
❌ Incomplete guides
❌ Plagiarized content
❌ Solution-focused
❌ No learning outcomes

## Development Setup

```bash
git clone https://github.com/via-decide/daxini-bca-learning.git
cd daxini-bca-learning
```

No dependencies needed! This is pure documentation.

## Commit Message Format

```
type: description

[optional body]

Examples:
- docs: add blockchain voting project guide
- fix: clarify database design section
- refactor: improve architecture template
```

Types: docs, fix, refactor, add, remove, update

## Pull Request Process

1. Create descriptive PR title
2. Reference related issues
3. Add description of changes
4. Wait for review
5. Address feedback
6. Merge when approved
