## Knowledge Gap Controls
- **Missing prerequisite knowledge** is handled by README navigation, learning paths, roadmaps, glossary material, and foundational guides.
- **Abrupt jumps in difficulty** are handled by project levels, module indexes, and repeated project document patterns.
- **Isolated documents** are assigned a role in the index below so they can be kept, expanded, or removed intentionally.
- **Duplicated explanations** are acceptable only when repeated in project-specific context; generic duplication should link back to shared guides.
- **Concepts introduced without explanation** should link to the closest foundation, architecture, API, database, deployment, or debugging document.

| Repository claim | Supporting documents | Validation |
|---|---|---|

| Understand production workflows | CI/CD, containers, monitoring, security, deployment, articles | Supported as conceptual and checklist-based workflow knowledge. |
| Think like an engineer instead of a tutorial follower | Philosophy, audit map, project planning docs, professional modules | Supported through reasoning-first structure and anti-copy-paste constraints. |
