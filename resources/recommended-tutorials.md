# Recommended Tutorials

This repository avoids copy-paste tutorials, but external learning resources are useful when chosen carefully. Prefer materials that explain concepts, constraints, and failure modes instead of only showing finished code.

## Selection Criteria

Choose resources that:

- Explain why a tool exists.
- Use official documentation or clearly state assumptions.
- Discuss errors, security, deployment, and maintenance.
- Encourage building small variations independently.

Avoid resources that:

- Provide complete projects without reasoning.
- Hide configuration and deployment details.
- Teach outdated or insecure practices without warning.

## How to Use Tutorials

1. Read the concept first.
2. Pause before implementation and draw your own design.
3. Build a different but related feature.
4. Document what changed and why.
