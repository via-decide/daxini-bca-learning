# Interview Prep

Interviews reward clear reasoning more than memorized definitions. Use your projects to prove that you can explain requirements, architecture, trade-offs, testing, and operations.

## Preparation Workflow

- Choose two projects you understand deeply.
- Draw each architecture from memory.
- Explain one database decision and one API decision.
- Prepare one debugging story and one trade-off story.
- Practice explaining to both technical and non-technical listeners.

## Common Question Types

- Why did you choose this architecture?
- How does data move through the system?
- What happens when the database is unavailable?
- How would you scale this?
- What would you improve next?
