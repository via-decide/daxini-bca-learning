# ⏱️ Time Tracking API: Learn By Building

**"Build a multi-user API where freelancers log hours against client projects, generating automated invoices and tracking real-time productivity."**

---

## 🎯 Learning Outcomes

After completing this project, you will understand:

✅ **Event-Based Architecture** - Managing running states (like timers) using Start/Stop events on the server rather than trusting the frontend's clock.  
✅ **SQL Date Math** - Calculating the difference between two timestamps (`end_time - start_time`) directly inside the database query.  
✅ **SQL Aggregation with JOINs** - Using `SUM()` and `GROUP BY` across multiple joined tables to generate financial reports (Invoices).  
✅ **Denormalization** - Knowing when to break strict database rules (like adding a redundant `user_id` to a child table) to massively improve read performance.

---

## 📋 Project Overview

### The Problem

Time tracking software (like Toggl or Harvest) must be perfectly accurate and immune to user tampering. If you trust the frontend browser to count seconds, you will lose data when a user closes their laptop. 

Furthermore, generating an invoice at the end of the month involves complex math: calculating the duration of hundreds of fragmented time entries, matching them to project hourly rates, and summing them up by Client, while ignoring entries that were already billed last month.

**Your job:** Build an API that securely manages the start/stop state of timers, and uses advanced SQL queries to generate accurate invoices.

### Who Uses It

```
Freelancer:
├─ POST /api/time/start (Begins tracking time)
├─ POST /api/time/:id/stop (Ends tracking time)
└─ GET /api/reports/unbilled (Generates the end-of-month invoice)
```

## 🧠 Implementation Strategy: Pseudocode

### 1. The Timer State Machine

The server controls time. 

```pseudocode
POST /api/time/start:
  middlewares: [authenticateUser]
  
  // 1. Check if they already have a timer running!
  active = db.query(`
    SELECT id FROM time_entries 
    WHERE user_id = ? AND end_time IS NULL
  `, req.user.id)
  
  if active.length > 0:
    return 409 "You already have a timer running!"
    
  // 2. Start the timer (CURRENT_TIMESTAMP is generated by the database engine safely)
  db.query(`
    INSERT INTO time_entries (id, project_id, user_id, start_time) 
    VALUES (UUID(), ?, ?, CURRENT_TIMESTAMP)
  `, [req.body.project_id, req.user.id])
  
  return 201 "Timer started"
```

### 2. Stopping the Timer (And calculating earnings)

```pseudocode
POST /api/time/:id/stop:
  middlewares: [authenticateUser]
  
  // 1. Update the end time to NOW
  db.query(`
    UPDATE time_entries 
    SET end_time = CURRENT_TIMESTAMP 
    WHERE id = ? AND user_id = ? AND end_time IS NULL
  `, [req.params.id, req.user.id])
  
  // 2. Fetch the calculated results to show the user
  // (Syntax varies slightly by DB, this is pseudocode)
  result = db.query(`
    SELECT 
      t.start_time, 
      t.end_time,
      (t.end_time - t.start_time) as duration_hours,
      (t.end_time - t.start_time) * p.hourly_rate as earned
    FROM time_entries t
    JOIN projects p ON t.project_id = p.id
    WHERE t.id = ?
  `, req.params.id)
  
  return 200 result
```

### 3. The Invoice Generator (Advanced SQL)

This is a beautiful example of letting the database do the heavy lifting. We want a report of all unbilled money, grouped by Client.

```pseudocode
GET /api/reports/unbilled:
  middlewares: [authenticateUser]
  
  // The God Query
  sql = `
    SELECT 
      c.id as client_id,
      c.name as client_name,
      SUM((t.end_time - t.start_time) / 3600.0) as total_hours,
      SUM(((t.end_time - t.start_time) / 3600.0) * p.hourly_rate) as total_owed
    FROM time_entries t
    JOIN projects p ON t.project_id = p.id
    JOIN clients c ON p.client_id = c.id
    WHERE t.user_id = ? 
      AND t.is_billed = 0 
      AND t.end_time IS NOT NULL  -- Ignore currently running timers
    GROUP BY c.id, c.name
  `
  
  report = db.query(sql, req.user.id)
  
  return 200 {
    message: "Unbilled Invoice Report",
    data: report
  }
```
