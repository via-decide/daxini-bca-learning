## Practical Examples

### Beginner Examples
1. Create a small sandbox project.
2. Add minimum configuration.
3. Document expected output.
4. Change one thing and validate behavior.
5. Share the result for review.

### Production Examples
1. Add automated validation.
2. Promote through staging before production.
3. Roll back a bad change safely.
4. Capture operational notes.
5. Review security and dependency risk.

### Enterprise Examples
1. Standardize policy across teams.
2. Integrate private infrastructure.
3. Audit changes and approvals.
4. Optimize for cost and reliability.
5. Migrate legacy workflows incrementally.

## Exercises

### Concept Questions
1. Explain the most important mental model in this module.
2. Describe a safe rollback strategy.
3. Identify when automation helps and when it hides risk.
4. Compare beginner and production workflows.
5. Explain how this module connects to one prerequisite.

### Practical Exercises
1. Build a minimal example.
2. Add validation.
3. Document setup and expected output.
4. Simulate a failure.
5. Recover and write a short postmortem.

### Debugging Exercises
1. Diagnose a broken configuration.
2. Resolve a conflicting change.
3. Find a missing dependency.
4. Restore a previous working state.
5. Improve performance or reliability.
